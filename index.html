✓ Paso 1 Preparar el sistema
👆 Haz clic en un comando para ver su explicación
# Actualizar todos los paquetes
sudo dnf update -y
▼
sudo ejecuta como administrador. dnf es el gestor de paquetes de CentOS Stream 10. update descarga e instala las versiones más recientes de todo lo instalado. -y acepta automáticamente todas las confirmaciones.
# Configurar firewall
sudo systemctl enable --now firewalld
▼
systemctl controla los servicios. enable hace que inicie automáticamente con cada reinicio. --now lo inicia también en este momento. firewalld es el servicio de firewall de CentOS.
sudo firewall-cmd --permanent --add-service=http
▼
--permanent guarda la regla de forma definitiva. --add-service=http abre el puerto 80 para tráfico web.
sudo firewall-cmd --permanent --add-service=https
▼
Abre el puerto 443 para tráfico HTTPS cifrado. Necesario si agregas SSL con Let's Encrypt.
sudo firewall-cmd --reload
▼
Aplica todas las reglas permanentes. Sin este comando los cambios no están activos aunque hayas usado --permanent.
sudo firewall-cmd --list-services
▼
Muestra los servicios abiertos. Debes ver http https para confirmar que las reglas quedaron aplicadas.
✓ Paso 2 Instalar Apache
👆 Haz clic en un comando para ver su explicación
sudo dnf install httpd -y
▼
httpd es el nombre del paquete de Apache en CentOS (HTTP Daemon = programa que corre en segundo plano atendiendo peticiones web).
sudo systemctl enable --now httpd
▼
Registra Apache para inicio automático y lo arranca inmediatamente en un solo comando.
sudo systemctl status httpd
▼
Muestra el estado del servicio. Busca active (running) en verde para confirmar que funciona.
sudo rm /etc/httpd/conf.d/welcome.conf
▼
Elimina la página de bienvenida por defecto de Apache que aparecía en lugar de tu sitio. Sin este archivo Apache sirve el contenido de tu carpeta directamente.
📁 Directorios importantes: /var/www/html/ raíz por defecto · /etc/httpd/conf.d/ Virtual Hosts · /var/log/httpd/ logs
✓ Paso 3 Instalar MySQL 8.4
⚠ Error que tuvimos — nombre incorrecto del paquete
sudo dnf install mysql-server -y
→ Error: No se pudo encontrar ningún resultado: mysql-server

sudo dnf install mysql8.4-server -y
→ Nombre correcto en CentOS Stream 10
👆 Haz clic en un comando para ver su explicación
sudo dnf install mysql8.4-server -y
▼
Instala MySQL 8.4. En CentOS Stream 10 el paquete incluye el número de versión en el nombre. Instala automáticamente dependencias como mysql8.4-common, mysql-selinux, mecab y otras.
sudo systemctl enable --now mysqld
▼
mysqld es el nombre del servicio de MySQL (la d = daemon). Registra para inicio automático y arranca inmediatamente.
sudo mysql_secure_installation
▼
Script interactivo de seguridad. Hace 6 preguntas para asegurar la instalación de MySQL. Ver sección siguiente para la explicación de cada pregunta.
✓ Paso 3b Preguntas de mysql_secure_installation
Pregunta 1 → N
Setup VALIDATE PASSWORD?
Activa reglas de complejidad para contraseñas. En práctica pon N, en producción Y.
Pregunta 2 → contraseña
New password for root
Contraseña del superusuario de MySQL. No tiene relación con el root de Linux.
Pregunta 3 → Y
Remove anonymous users?
Elimina usuario vacío sin contraseña que MySQL crea por defecto. Siempre Y.
Pregunta 4 → Y
Disallow root login remotely?
Impide conectarse como root desde otra máquina. Para acceso remoto usa usuarios con permisos limitados.
Pregunta 5 → Y
Remove test database?
Borra la base de datos test que cualquier usuario puede ver. No sirve en producción.
Pregunta 6 → Y
Reload privilege tables?
Aplica los cambios en memoria sin reiniciar. Si dices N los cambios no tienen efecto hasta el próximo reinicio.
✓ Paso 3c Crear base de datos y usuario
⚠ Errores que tuvimos en MySQL
CREATE USER miapp_user@localhost IDENTIFIED BY Contrasena123!
→ ERROR 1064: faltan comillas simples ' '

CREATE DATABSE miapp_db;
→ ERROR 1064: typo — falta la A en DATABASE

Si ves -> en lugar de mysql>
→ El comando está incompleto, escribe \c y Enter para cancelar
👆 Haz clic en un comando para ver su explicación
sudo mysql -u root -p
▼
mysql es el cliente de línea de comandos. -u root entra como superusuario de MySQL. -p pide la contraseña de forma segura.
DROP DATABASE IF EXISTS miapp_db;
▼
DROP DATABASE elimina la base de datos. IF EXISTS evita error si no existía. El ; es OBLIGATORIO — sin él MySQL espera más instrucciones y muestra ->.
CREATE DATABASE miapp_db;
▼
Crea la base de datos donde vivirán todas las tablas de la aplicación.
CREATE USER 'user1'@'localhost' IDENTIFIED BY 'Contrasenasegura123!';
▼
Las comillas simples ' ' son OBLIGATORIAS para delimitar texto. Sin ellas MySQL lanza ERROR 1064. @'localhost' significa que solo puede conectarse desde el mismo servidor.
GRANT ALL PRIVILEGES ON miapp_db.* TO 'user1'@'localhost';
▼
GRANT ALL PRIVILEGES da todos los permisos: SELECT, INSERT, UPDATE, DELETE. miapp_db.* = todas las tablas de esa base de datos.
FLUSH PRIVILEGES;
▼
Aplica los permisos en memoria inmediatamente sin reiniciar MySQL.
EXIT;
▼
Sale del cliente MySQL y regresa al terminal de Linux.
✓ Paso 4 Instalar PHP
👆 Haz clic en un comando para ver su explicación
sudo dnf install php php-mysqlnd php-fpm php-json php-common php-mbstring php-xml php-gd php-curl -y
▼
php intérprete base. php-mysqlnd driver nativo de MySQL — sin él PHP no puede conectarse a la BD. php-fpm gestor de procesos PHP para Apache. php-json maneja JSON. php-mbstring soporta acentos y caracteres especiales. php-xml procesa XML. php-gd manipula imágenes. php-curl hace peticiones HTTP.
sudo systemctl enable --now php-fpm
▼
En CentOS Stream 10 PHP usa php-fpm en lugar de cargarse directamente en Apache. Sin este servicio activo Apache no puede procesar archivos .php y devuelve "File not found".
sudo systemctl restart httpd
▼
Reinicia Apache para que cargue el módulo de PHP recién instalado.
⚠ Lección aprendida: en CentOS Stream 10 php-fpm debe estar activo para que PHP funcione. Si ves "File not found" al abrir un .php, verifica con sudo systemctl status php-fpm
✓ Paso 5 Virtual Host
👆 Haz clic en un comando para ver su explicación
sudo mkdir -p /var/www/misitio.com/html
▼
mkdir -p crea el directorio y todos los directorios padre necesarios en un solo comando.
sudo chown -R $USER:$USER /var/www/misitio.com
▼
chown cambia el propietario. -R aplica recursivamente. $USER es tu usuario actual.
sudo chmod -R 755 /var/www/misitio.com
▼
755: propietario puede leer/escribir/ejecutar. Grupo y otros solo leer y ejecutar. Apache puede leer los archivos pero no modificarlos.
sudo apachectl configtest
▼
Verifica la sintaxis de todos los archivos de configuración sin reiniciar Apache. Siempre ejecutar antes de recargar.
sudo systemctl reload httpd
▼
Recarga la configuración sin interrumpir conexiones activas. Úsalo después de cambiar archivos .conf.
/etc/httpd/conf.d/misitio.com.conf
<VirtualHost *:80> ServerName misitio.com ServerAlias www.misitio.com DocumentRoot /var/www/misitio.com/html ErrorLog /var/log/httpd/misitio-error.log CustomLog /var/log/httpd/misitio-access.log combined <Directory /var/www/misitio.com/html> AllowOverride All Require all granted </Directory> </VirtualHost>
💡 Lección aprendida: cuando solo hay un Virtual Host, Apache lo usa para TODAS las peticiones incluyendo las que llegan por IP directa.

✓ SELinux Permisos para PHP y MySQL
👆 Haz clic en un comando para ver su explicación
sudo setsebool -P httpd_can_network_connect 1
▼
setsebool cambia valores booleanos de SELinux. -P hace el cambio permanente. httpd_can_network_connect permite que Apache y PHP hagan conexiones de red incluyendo a MySQL.
sudo setsebool -P httpd_can_network_connect_db 1
▼
Permite específicamente que Apache y PHP se conecten a bases de datos como MySQL. Necesario para que PDO funcione correctamente.
sudo restorecon -R /var/www/html/
▼
Restaura los contextos de seguridad SELinux en los archivos web. Necesario cuando copias o mueves archivos que Apache necesita leer.
Ref Comandos de mantenimiento
👆 Haz clic en un comando para ver su explicación
sudo systemctl status httpd mysqld php-fpm
▼
Muestra el estado de los tres servicios del stack LAMP. Busca active (running) en los tres.
sudo systemctl restart httpd mysqld php-fpm
▼
Reinicia los tres servicios. Úsalo cuando instalas nuevas extensiones o cambias configuraciones.
sudo tail -f /var/log/httpd/error_log
▼
tail -f muestra las últimas líneas en tiempo real. El primer lugar donde buscar cuando algo falla.
sudo tail -f /var/log/php-fpm/error.log
▼
Log de errores de PHP-FPM. Útil cuando Apache funciona pero PHP da errores.
httpd -v && php -v && mysql --version
▼
Muestra las versiones de Apache, PHP y MySQL en un solo comando.
★ Lecciones aprendidas en clase
⚡ Errores y soluciones que vivimos
El paquete MySQL se llama mysql8.4-server, NO mysql-server en CentOS Stream 10
Las comillas simples ' ' son OBLIGATORIAS en MySQL para delimitar texto
Siempre terminar comandos MySQL con ; — si ves -> escribe \c para cancelar
En CentOS Stream 10 PHP usa php-fpm — debe estar activo o Apache devuelve "File not found"
El botón de formulario PHP necesita value="1" y usar isset() para detectar el POST
Si hay un solo Virtual Host, Apache lo usa para TODAS las peticiones incluyendo las por IP
Al copiar archivos a /var/www/ ejecutar sudo restorecon -R para permisos SELinux
Usar comillas simples ' ' con echo en bash cuando el texto tiene ! para evitar errores
